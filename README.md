# Smart Glass Medical Assistant

A capstone project from Penn State (Group 207) that pairs smart-glass OCR hardware with an AI-powered backend to help clinicians query, summarize, and converse about medical documentation hands-free. The system captures PDF content in the user's field of view, processes it through specialized language models, and returns structured answers with document citations.

## System Architecture

The backend is a **FastAPI** application (`webapi/`) that auto-discovers and registers endpoint modules at startup. All requests and responses share a common structure (`reqRefId`, `resRefId`, `prompt`) defined in `utils/req_res_structures.py`.

## API Endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/advanced/query` | POST | Medical Q&A with optional PDF upload; returns cited `DocumentReference` objects |
| `/summarize/` | POST | Abstractive summarization via LongT5 with optional word-count cap |
| `/convo/` | POST | Stateful multi-turn conversation; set `new=true` to reset history |
| `/in-context` | POST | In-context learning inference |

### Document-Grounded Query (`/advanced/query`)

This is the primary endpoint for the smart-glass use case. It accepts a multipart form with an optional PDF and a `highlights` field (JSON array of `{word, pages}` objects) that scopes the answer to specific regions the user has highlighted in view. When a PDF is provided, inference runs in two phases and the response includes page numbers, verbatim quotes, and confidence scores for each reference.

## Models

- **MedGemma** (`google/medgemma`) - Handles general medical Q&A, drug-interaction extraction (structured JSON output), and multi-turn conversation.
- **LongT5** (`google/long-t5-tglobal-base`) - Handles abstractive summarization with a 16,384-token context window.

Both models are loaded as singletons at process startup via `webapi/inference/medgemma_utils.py` to avoid redundant loading across requests.

## Getting Started

Install dependencies and run the server:

```bash
pip install -r webapi/requirements.txt
cd webapi
python main.py
```

The API will be available at `http://localhost:8000`. Interactive docs are auto-generated by FastAPI at `/docs`.

### AWS Deployment

A separate `webapi/aws_requirements.txt` is provided for deploying to an AWS environment.

## Testing

Tests are organized into three layers under `webapi/tests/`:

- `unit/` - Isolated tests for inference wrappers, utilities, and schema validation
- `component/` - Tests for individual endpoint behavior
- `func_req/` - End-to-end functional requirement tests

Run all tests from the `webapi/` directory:

```bash
python -m pytest tests
```
